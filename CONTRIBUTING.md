# Contributing to WNH-INVENTORY

Welcome to the lab. This inventory is an open, peer-to-peer tracker. Whether you just pulled a broken router from a bin or you are taking an unallocated smartphone to build a custom Linux node, this is how you document it.

## ⚠️ The Golden Rule: DO NOT BLOAT THE REPO

We store original hardware photos in this repository, which means it can bloat to gigabytes in weeks if we aren't careful. **You must compress your images before committing.**

We have a `make` command to handle this automatically using ImageMagick:
1. Drop your raw smartphone photos into the correct hardware folder.
2. Run `make compress` in your terminal.
3. Commit the changes.

*Pull requests containing raw 5MB+ image files will be rejected.*

---

## 📥 Adding New Hardware

To log a new piece of intercepted e-waste:

1. Fork this repository and create a new branch.
2. Run the intake script from the root directory:
   ```bash
   ./new_donation.sh
   ```
3. Follow the prompts. The script will automatically generate the correct directory structure (`content/hardware/wnh-XXX-slug/`) and assign the next available `asset_id`.
4. Place your compressed "before" photos into the newly created `initial/` directory.
5. Open the generated `index.md` and fill out the Technical Overview and Rescue Log.

---

## 🛠️ Updating Existing Hardware (Claiming a Device)

If you are taking an `unallocated` device from the lab to work on it, you must update the inventory to reflect your operator status.

1. Find the device's `index.md` file in `content/hardware/`.
2. Update the YAML Front Matter:
   * Change `status` to `Active Compute Node`.
   * Add your name to `operator`.
   * Add a link to your GitHub/LinkedIn in `operator_link`.
3. Append a new update to the `updates` array to log what you are doing:
   ```yaml
   updates:
     - date: "2026-04-22"
       note: "Claimed device. Bypassing the battery to wire directly to 5V."
   ```
4. If you have progress photos, drop the compressed images into the `current/` folder. The UI will automatically detect them and build a "Current Build" grid.

## 📋 Front Matter Schema Reference

Do not alter the YAML keys. Only modify the values.

```yaml
title: "Device Name"            # Keep it short (e.g., TL-MR3020)
asset_id: "WNH-001"             # Auto-generated by the script
class: "Router"                 # Broad category
architecture: "MIPS"            # CPU Arch (ARM, x86, MIPS, etc.)
ram: "32MB"                     # Optional
status: "unallocated"           # MUST BE: "unallocated", "Active", or "Awaiting Triage"
donor: "John Doe"               # Who found/donated it
donor_link: "https://..."       # Optional link
operator: "Jane Doe"            # Who is currently hacking it
operator_link: "https://..."    # Optional link
date: 2026-04-21T17:51:28+01:00 # Auto-generated
updates:                        # Array of progress logs
  - date: "YYYY-MM-DD"
    note: "Brief description of the work done."
```

## 🚢 Submitting Your PR

Once your Markdown is written and your images are compressed:
1. Push your branch to your fork.
2. Open a Pull Request against the `main` branch of `WNH-Organization/WNH-INVENTORY`.
3. Once merged, GitHub Actions will automatically compile the SSG and deploy the changes to the live URL within seconds.
